# BPMN — Quy trình sử dụng OpenCode PrivateGPT Bridge MVP

## Purpose

Mô tả quy trình nghiệp vụ để developer dùng OpenCode CLI với OpenCode PrivateGPT Bridge, từ khởi động/xác thực/sync Agent đến khi nhận response từ PrivateGPT. Quy trình giúp thiết kế, kiểm thử và xác nhận rằng Bridge là integration boundary, còn OpenCode vẫn sở hữu coding workflow, thao tác file, command và approval.

## Context

Phạm vi áp dụng là MVP được mô tả trong `00.requirement.md`: Bridge Python chạy local, chỉ expose `gemini-2.5-pro`, bắt buộc dùng PrivateGPT Agent API và chuyển request OpenAI-compatible của OpenCode sang PrivateGPT Conversation API. LiteLLM, nhiều model, tool execution trong Bridge, IntelliJ runtime và lưu full chat history không thuộc luồng này.

**Mục tiêu quy trình:** một developer đã cài đặt/cấu hình cần thiết có thể khởi động Bridge, hoàn tất login khi cần, đồng bộ Agent theo workspace và nhận response PrivateGPT qua OpenCode mà Bridge không thực thi tool.

**Trigger:** developer chạy `privategpt-adapter start` để chuẩn bị Bridge cho OpenCode; sau khi Bridge sẵn sàng, OpenCode gửi `POST /v1/chat/completions` cho một tác vụ coding.

**Điều kiện tiền đề được mô tả:** Bridge chạy local, cấu hình và secure token store có thể được kiểm tra; OpenCode được cấu hình dùng endpoint local. Requirement chưa mô tả đầy đủ điều kiện trước khi chạy `install` hoặc cách provision cấu hình ban đầu.

## Participants

| Participant | Responsibility |
|---|---|
| Developer | Khởi động/login Bridge, cấu hình OpenCode và phê duyệt các thao tác coding do OpenCode đề xuất. |
| OpenCode CLI | Gửi request OpenAI-compatible; quản lý repository context, file read/edit, command execution và permission. |
| OpenCode PrivateGPT Bridge | Kiểm tra config/auth/model, đồng bộ Agent, map request/response, gọi PrivateGPT và parse SSE. Không thực thi tool hoặc thao tác workspace. |
| Secure Token Store | Lưu refresh token an toàn và hỗ trợ Bridge kiểm tra token state; access token được Bridge cache trong memory. |
| PrivateGPT Model API | Cung cấp danh sách support model để Bridge resolve internal model ID. |
| PrivateGPT Agent API | List/create/update/get Agent server-side cho workspace. |
| PrivateGPT Conversation API | Nhận PrivateGPT ChatRequest có `metadata.agent_id` và trả response SSE. |
| Gemini 2.5 Pro | Model backend duy nhất được expose trong MVP thông qua PrivateGPT. |

## Happy Path

### BPMN model summary

| Element | Type | Description |
|---|---|---|
| Start event | Message/start event | Developer chạy `privategpt-adapter start`. |
| Intermediate message event | Browser login callback | User hoàn tất login; callback trả về localhost khi login là cần thiết. |
| Intermediate message event | PrivateGPT SSE | Conversation API trả stream response cho Bridge. |
| End event | Success | OpenCode nhận response OpenAI-compatible và tiếp tục coding workflow dưới quyền/approval của OpenCode. |

### Main flow

1. **Start event — Khởi động Bridge**
   - **Performer:** Developer.
   - **Input:** Lệnh `privategpt-adapter start`; local configuration.
   - **Business action:** Developer yêu cầu Bridge chuẩn bị local server cho OpenCode.
   - **Output:** Quy trình startup được kích hoạt.
   - **Next:** Bridge tải configuration và kiểm tra điều kiện local.

2. **Task — Tải cấu hình và kiểm tra điều kiện khởi động**
   - **Performer:** OpenCode PrivateGPT Bridge.
   - **Input:** Adapter config, token-store access, port availability theo doctor/start requirements.
   - **Business action:** Kiểm tra auth status, secure token store, model mapping và các điều kiện để chạy local server.
   - **Output:** Trạng thái sẵn sàng để kiểm tra login/model/Agent hoặc lỗi chẩn đoán rõ ràng.
   - **Next:** Gateway “Có login hợp lệ?”.

3. **Gateway — Có login hợp lệ?**
   - **Type:** Exclusive gateway.
   - **Condition yes:** Bridge có token/session hợp lệ; tiếp tục resolve model.
   - **Condition no:** Developer phải login qua browser; chuyển sang task login.

4. **Task — Login PrivateGPT qua browser**
   - **Performer:** Developer và OpenCode PrivateGPT Bridge.
   - **Input:** Lệnh `privategpt-adapter login`; browser login flow.
   - **Business action:** Bridge mở browser; developer đăng nhập; OAuth callback trở về localhost; Bridge đổi authorization code lấy token.
   - **Output:** Refresh token được lưu an toàn, access token được cache in-memory, trạng thái login có thể report.
   - **Next:** Bridge kiểm tra khả năng gọi PrivateGPT rồi quay lại resolve model.

5. **Task — Resolve Gemini 2.5 Pro internal model ID**
   - **Performer:** OpenCode PrivateGPT Bridge và PrivateGPT Model API.
   - **Input:** Public model alias `gemini-2.5-pro`; support-models response hoặc explicit configuration.
   - **Business action:** Bridge chỉ resolve model alias được whitelist; không thay model khác.
   - **Output:** Internal model ID hoặc lỗi `model_not_available`/`model_not_supported`.
   - **Next:** Gateway “Model được resolve?”.

6. **Gateway — Model được resolve?**
   - **Type:** Exclusive gateway.
   - **Condition yes:** Có internal model ID cho Gemini 2.5 Pro; tiếp tục sync Agent.
   - **Condition no:** Trả lỗi rõ ràng, không fallback model; kết thúc lỗi.

7. **Task — Đồng bộ PrivateGPT Agent theo workspace**
   - **Performer:** OpenCode PrivateGPT Bridge và PrivateGPT Agent API.
   - **Input:** Workspace hash, internal model ID, OpenCode-compatible instruction, `instructions_hash`, agent metadata cache.
   - **Business action:** List Agent hiện có; tạo Agent nếu thiếu; update nếu model ID hoặc instruction hash thay đổi; cache `agent_id`, model ID, instruction hash và thời điểm update.
   - **Output:** Agent đã đồng bộ có `agent_id`, hoặc lỗi sync rõ ràng.
   - **Next:** Gateway “Agent sync thành công?”.

8. **Gateway — Agent sync thành công?**
   - **Type:** Exclusive gateway.
   - **Condition yes:** Bridge có `agent_id`; khởi động local OpenAI-compatible server và báo ready cho OpenCode.
   - **Condition no:** Không fallback sang inline instruction; trả lỗi sync và kết thúc lỗi.

9. **Task — OpenCode gửi chat completion**
   - **Performer:** OpenCode CLI.
   - **Input:** `POST /v1/chat/completions`, model `gemini-2.5-pro`, OpenCode messages, lựa chọn `stream`.
   - **Business action:** OpenCode gửi tác vụ coding tới local Bridge sau khi Bridge ready.
   - **Output:** OpenAI-compatible request được gửi đến Bridge.
   - **Next:** Bridge validate request/model/auth.

10. **Task — Validate request, token và Agent binding**
    - **Performer:** OpenCode PrivateGPT Bridge.
    - **Input:** OpenCode request, public model, token state, cached/synchronized Agent metadata.
    - **Business action:** Validate model/request; refresh access token trước expiry và retry forced refresh tối đa một lần khi HTTP 401; đảm bảo `agent_id` hợp lệ.
    - **Output:** Request đủ điều kiện để map, hoặc lỗi authentication/model/Agent rõ ràng.
    - **Next:** Gateway “Request có hợp lệ và auth/Agent sẵn sàng?”.

11. **Gateway — Request có hợp lệ và auth/Agent sẵn sàng?**
    - **Type:** Exclusive gateway.
    - **Condition yes:** Map request sang PrivateGPT ChatRequest.
    - **Condition no:** Trả lỗi OpenAI-compatible; lỗi auth dẫn developer chạy login, lỗi model không fallback, lỗi Agent không dùng inline instruction; kết thúc lỗi.

12. **Task — Build PrivateGPT ChatRequest và gọi Conversation API**
    - **Performer:** OpenCode PrivateGPT Bridge và PrivateGPT Conversation API.
    - **Input:** Rendered OpenCode messages, internal model ID, `metadata.agent_id`, `tools: []`, `conversation_id: null`, `parent_message_id: null` trong MVP.
    - **Business action:** Bridge giữ system/developer messages, map request theo Agent Mode và gửi conversation request bằng Bearer token.
    - **Output:** SSE response stream hoặc lỗi upstream.
    - **Next:** Gateway “OpenCode yêu cầu streaming?”.

13. **Gateway — OpenCode yêu cầu streaming?**
    - **Type:** Exclusive gateway theo field `stream` của request.
    - **Condition yes:** Bridge relay OpenAI-compatible SSE chunks.
    - **Condition no:** Bridge thu visible assistant content và tạo non-stream OpenAI-compatible response.

14. **Task — Parse và map response PrivateGPT**
    - **Performer:** OpenCode PrivateGPT Bridge.
    - **Input:** SSE event stream từ PrivateGPT; request correlation context.
    - **Business action:** Thu visible assistant content; lọc reasoning marker nội bộ nếu có; capture conversation/message ID nếu event cung cấp; dừng ở DONE/stream end; không trả plugin tool call như hành động thực thi.
    - **Output:** SSE chunks và `[DONE]` nếu `stream=true`, hoặc OpenAI-compatible non-stream response nếu `stream=false`.
    - **Next:** OpenCode nhận response.

15. **End event — OpenCode nhận response và tiếp tục coding workflow**
    - **Performer:** OpenCode CLI.
    - **Input:** OpenAI-compatible response từ Bridge.
    - **Business action:** OpenCode hiển thị/tiếp tục workflow; file operation, command execution và approval vẫn do OpenCode quản lý.
    - **Output:** Kết quả phản hồi cho developer; không có tool execution trong Bridge.

### Data objects and messages

| Data object / message | Created or supplied by | Used by | Lifecycle / content stated in requirement |
|---|---|---|---|
| Adapter configuration | `install`/local configuration | Bridge startup and doctor | Host, port, PrivateGPT URL, model, auth, Agent, OpenAI compatibility, logging, streaming/resilience settings. |
| Refresh token / access token | Browser login and token flow | Bridge / Secure Token Store / PrivateGPT API | Refresh token stored securely; access token cached in memory; neither logged. |
| Support-models response | PrivateGPT Model API | Model resolver | Used to resolve internal Gemini 2.5 Pro model ID. Exact response schema remains missing information. |
| Workspace Agent metadata | Agent API and Bridge | Agent sync and chat request | `agent_id`, `model_id`, `instructions_hash`, `updated_at`; avoids recreating Agent per request. |
| OpenCode-compatible instruction | Bridge | PrivateGPT Agent API | Versioned/hashed instruction; must not require IntelliJ plugin tool-call protocol. |
| OpenAI-compatible request | OpenCode CLI | Bridge | Model, messages and `stream`; actual OpenCode payload is a validation condition. |
| PrivateGPT ChatRequest | Bridge | PrivateGPT Conversation API | Contains internal model ID, `metadata.agent_id`, `tools: []`; no global conversation ID in MVP. |
| PrivateGPT SSE response | Conversation API | Bridge response mapper | Exact event schema remains missing information. |
| OpenAI-compatible response | Bridge | OpenCode CLI | Non-stream response or SSE chunks followed by `[DONE]` when streaming. |
| Redacted diagnostic/log record | Bridge | Developer/operations | Must not include token, cookie, raw prompt or raw source code. |

## Error Or Alternate Paths

| Path | Trigger / gateway condition | Handling stated in requirement | End state |
|---|---|---|---|
| Login required | No valid login, refresh failure or session expired | Report login-required/token-refresh-failed; developer runs `privategpt-adapter login`. | Error/return-to-login path. |
| Token refresh retry | HTTP 401 while ensuring token | Force refresh and retry once. | Continue if successful; otherwise login-required error. |
| Unsupported model | Requested model is not `gemini-2.5-pro` or internal mapping fails | Reject with clear model error; do not fallback to another model. | Error end. |
| Agent synchronization failure | Agent cannot be list/create/update/get or sync fails | Fail clearly; do not silently use inline instruction. | Error end. |
| Unsupported tool payload | OpenCode sends tool requirement that cannot be safely ignored | Reject clearly; adapter does not execute tools. | Error end; compatibility spike captures the actual need. |
| Plugin-style tool call in model response | PrivateGPT Agent emits `<tool_call>` | Detect and return `privategpt_agent_instruction_mismatch`; do not execute it; reset/sync Agent as mitigation. | Error end. |
| Upstream timeout/unreachable/rate limit before first response chunk | PrivateGPT connection/response fails before relay | Retry is permitted only before the first relayed chunk under bounded resilience policy. | Continue if retry succeeds; otherwise mapped error end. |
| Failure after first relayed chunk | PrivateGPT stream fails after client-facing output | Do not retry whole request to avoid duplicate output. | Stream/error end. |
| Client disconnect or cancellation | OpenCode stops request/user cancels | Close upstream stream and release connection; do not leave background work running. | Cancelled end. |
| Conversation context contamination risk | Reuse of a global conversation ID | MVP uses `conversation_id = null`; future session mapping must be workspace/session keyed with TTL. | Main flow remains isolated. |

## Các câu hỏi cần làm rõ với khách hàng

### Actors và trách nhiệm

1. Ai chịu trách nhiệm provision cấu hình ban đầu, secure token store và OpenCode provider configuration trước khi developer chạy `start`?
2. “PrivateGPT Platform/API” có cần được xem là một participant nghiệp vụ duy nhất hay các Model/Agent/Conversation API cần được khách hàng xác nhận là các external system/lane riêng trong BPMN chính thức?

### Task và gateway

1. Trigger nghiệp vụ chính thức là developer chạy `start`, developer gửi một task qua OpenCode, hay cả hai cần là hai process BPMN riêng?
2. Agent status hoặc điều kiện nào được PrivateGPT xem là “ready” sau create/update?
3. Các bước `install`, `stop`, `restart`, `logout`, `agent reset` có cần là process BPMN độc lập hay là alternate path của process này?

### Data object và system integration

1. Internal model ID chính xác cho Gemini 2.5 Pro là gì; dùng discovery, cấu hình explicit, hay cả hai theo thứ tự nào?
2. JSON schema runtime của support-models, Agent create/update và PrivateGPT ChatRequest là gì?
3. SSE event names/payload nào biểu thị visible content, reasoning marker, conversation ID, message ID, done và error?
4. OpenCode payload thực tế có gửi `tools`, `tool_choice`, `tool_call_id` hoặc tool-result message không?
5. Phiên bản/payload OpenCode có bắt buộc `stream=true`, và có yêu cầu native OpenAI tool calls không?

### Exception flow và business rule

1. Error response/exit behavior nào khách hàng mong đợi cho timeout, rate limit, PrivateGPT unreachable và failure sau khi đã relay chunk đầu tiên?
2. Timeout, max accumulated response size, retry budget và circuit-breaker threshold cần được xác nhận ở mức business/operational policy nào?
3. Khi OpenCode yêu cầu tool calling không được hỗ trợ, khách hàng xác nhận reject toàn request hay cho phép bỏ qua an toàn trong trường hợp nào?

### Security và dữ liệu

1. Cơ chế secure token store nào được phê duyệt cho môi trường developer mục tiêu?
2. Chính sách PrivateGPT về việc gửi source code nội bộ, retention và logging là gì?
3. Có cần approval/audit bổ sung ngoài permission mặc định của OpenCode trước khi OpenCode đọc file, đề xuất edit hoặc chạy command không?
