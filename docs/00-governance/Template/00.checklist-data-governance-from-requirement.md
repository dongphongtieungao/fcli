# Data Governance From Requirement Checklist

## Purpose

Check whether a requirement introduces data governance work.

## Context

Run this checklist before promoting a data-related requirement into implementation.

## Checklist

- [ ] Data sources are identified.
- [ ] Data owners are identified or marked `unknown`.
- [ ] Sensitivity is classified.
- [ ] Retention expectations are known.
- [ ] Access controls are known.
- [ ] Quality checks are defined.
- [ ] Logging and audit needs are known.
