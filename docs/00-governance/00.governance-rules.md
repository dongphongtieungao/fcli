# Governance Rules

## Purpose

Define reusable governance rules for documentation, source-of-truth control, agent work, verification, and promotion.

## Context

These rules are intentionally generic. Add project-specific rule IDs only when a new project decision, risk, or recurring review finding justifies them.

## Core Rules

| ID | Rule | Severity | Verification |
|---|---|---|---|
| GOV-001 | Official docs must not invent source-of-truth evidence. | Major | Documentation review |
| GOV-002 | Secrets and sensitive data must not be committed. | Blocker | Review or secret scan |
| GOV-003 | Behavior changes should have matching tests or verification notes. | Major | Test/evidence review |
| GOV-004 | New durable docs should be linked from related artifacts. | Minor | Link review |
| GOV-005 | Workspace notes must not override official docs. | Major | KMS review |
| GOV-006 | Registry files must remain populated and parseable. | Major | Manifest/source catalog parse and path check |
| GOV-007 | Promotion must record source, target, sync status, and unpromoted content. | Major | `promotion.md` review |
| GOV-008 | Verification claims must name the evidence used or the reason evidence is unavailable. | Major | QA/DoD review |

## Decision Rules

| Situation | Required Action |
|---|---|
| Source conflict | Stop and record the conflict; do not silently choose a winner. |
| Missing source | Mark the gap and continue only with safe analysis or draft work. |
| Durable observation | Promote to the correct docs layer or record `SYNC_NEEDED`. |
| Sensitive data risk | Redact, remove, or block until approved handling exists. |
| Release decision | Require evidence and human owner; agent cannot self-approve release. |

## Promotion Rules

Promote a governance rule when it applies to more than one task, prevents a repeated mistake, or controls source-of-truth behavior. Do not promote one-off preferences as governance unless the user explicitly decides they are policy.
