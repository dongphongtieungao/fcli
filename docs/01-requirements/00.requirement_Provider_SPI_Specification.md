# Provider SPI Specification

## Purpose

Define the asynchronous provider boundary used by the OpenCode PrivateGPT Bridge. The SPI isolates the OpenAI-compatible API and Agent Service from PrivateGPT internal HTTP details while preserving a narrow, testable MVP surface.

## Appendix Relationship

This is the Provider SPI contract appendix of [the canonical requirement](./00.requirement.md). It defines only the provider boundary needed to satisfy that requirement. It cannot add client-facing endpoints, alter business acceptance, or commit a future provider/capability without a parent-requirement change.

## MVP Profile

| Topic | Contract |
|---|---|
| Implemented provider | `PrivateGPTProvider` only. |
| Public model | `privategpt/gemini-2.5-pro`; the client-facing alias remains `gemini-2.5-pro`. |
| Execution mode | Agent API is mandatory; every conversation contains the synchronized `metadata.agent_id`. |
| Transport | Async HTTP to PrivateGPT; incoming and outgoing streaming uses SSE. |
| Tool boundary | `tool_calling=False`; providers do not execute tools. |
| Extension | A future provider may be added only by implementing this SPI and declaring capabilities. It is not an MVP commitment. |

## Architectural Position

```text
OpenCode CLI
  -> OpenAI-compatible API layer
  -> Gateway / Agent Service
  -> Provider SPI
  -> PrivateGPTProvider
  -> PrivateGPT internal Model, Agent and Conversation APIs
```

The API layer and Agent Service depend on SPI abstractions only. A concrete provider must never be imported directly by an API route or business-service module.

## Capability Contract

```python
@dataclass(frozen=True)
class ProviderCapabilities:
    streaming: bool
    agent_mode: bool
    tool_calling: bool
    system_messages: bool
    supported_models: tuple[str, ...]
```

`PrivateGPTProvider` must declare `streaming=True`, `agent_mode=True`, `tool_calling=False`, and exactly one supported public model: `gemini-2.5-pro`.

## Unified Data Contracts

The API layer maps an OpenAI-compatible request into `UnifiedChatRequest`; it must not pass raw HTTP payloads into a provider.

```python
@dataclass(frozen=True)
class UnifiedChatRequest:
    public_model: str
    messages: Sequence[UnifiedMessage]
    stream: bool
    metadata: Mapping[str, object]

@dataclass(frozen=True)
class AgentBinding:
    workspace_key: str
    agent_id: str
    internal_model_id: str
    instructions_hash: str
```

`metadata` is adapter-owned metadata. The provider converts `AgentBinding.agent_id` into the provider request's `metadata.agent_id`; it must not expose credentials, cookies, or local filesystem paths.

For a non-stream request, the provider returns a `UnifiedChatResponse` with visible assistant content, finish reason, optional usage, and safe provider metadata. It must not return a raw PrivateGPT response or secret-bearing headers.

## Provider Interface

```python
class ProviderAdapter(Protocol):
    async def initialize(self) -> None: ...
    async def health_check(self) -> ProviderHealth: ...
    async def list_models(self) -> Sequence[ProviderModel]: ...
    async def resolve_model(self, public_model: str) -> ProviderModel: ...
    async def ensure_agent(self, request: AgentSyncRequest) -> AgentBinding: ...
    async def complete(self, request: UnifiedChatRequest, agent: AgentBinding) -> UnifiedChatResponse: ...
    def stream(self, request: UnifiedChatRequest, agent: AgentBinding) -> AsyncIterator[ProviderStreamEvent]: ...
```

`ensure_agent` is idempotent for the workspace key. It creates or updates the agent only when it is missing, its internal model ID changes, or its instruction hash changes. Failure to synchronize the agent is terminal for the request; inline-instruction fallback is prohibited.

## Streaming Contract

`stream()` yields typed events rather than raw string fragments:

```python
class ProviderStreamEventType(StrEnum):
    DELTA = "delta"
    COMPLETED = "completed"
    ERROR = "error"

@dataclass(frozen=True)
class ProviderStreamEvent:
    type: ProviderStreamEventType
    text: str | None = None
    finish_reason: str | None = None
    error: ProviderError | None = None
```

The OpenAI response mapper owns framing these events as SSE chunks and `[DONE]`. The provider owns parsing upstream PrivateGPT SSE only. Once the mapper relays the first delta, the request is non-retriable.

On client cancellation, the gateway cancels iteration and the provider closes the upstream response. It must not keep a detached generation running or emit more chunks.

## Errors, Retry and Circuit Breaking

Providers raise only `ProviderError` with a safe code, category, retryability and redacted message.

| Condition | Provider outcome | Gateway policy |
|---|---|---|
| Missing or expired session | Authentication error | Return login-required error; no conversation retry. |
| Unsupported model | Validation error | Reject without fallback. |
| Connect/timeout/429/5xx before first delta | Transient error | Retry may be considered by the resilience layer. |
| Failure after first delta | Stream error | End the response safely; do not retry. |
| Agent synchronization failure | Agent error | Return a clear error; do not use inline instruction. |

Retry and circuit-breaking are cross-cutting gateway responsibilities, not provider-local policy. A provider may expose enough classification for that policy but must not create hidden duplicate retries.

## Registration and Resolution

The registry maps a provider-qualified model to a provider adapter. In the MVP, the only resolvable qualified model is `privategpt/gemini-2.5-pro`. The registry validates the public model whitelist before any external request.

Future additions require a contract test proving that the API layer and Agent Service remain unchanged, plus an explicit architecture decision when their capability set or authentication model differs.

## Verification

| Contract area | Evidence required before implementation is claimed complete |
|---|---|
| Model resolution | Captured/redacted model-discovery response and unsupported-model test. |
| Agent synchronization | Create/update evidence proving idempotency and `metadata.agent_id` propagation. |
| Non-stream response | Mock or runtime conversation response mapped to the unified response. |
| SSE | Redacted upstream event capture, SSE mapper test, cancellation test and no-retry-after-first-delta test. |
| Boundary | Dependency review proving API/Agent code has no direct concrete-provider or HTTP-client dependency. |
