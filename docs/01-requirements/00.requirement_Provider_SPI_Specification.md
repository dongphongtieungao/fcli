# Provider SPI Specification
## PrivateGPT Adapter Platform

Version: 1.0
Status: Published
Audience: Adapter Developers, Provider Developers

## Parent Requirement

This published appendix defines the Provider SPI contract for the canonical [PrivateGPT Adapter requirement](./00.requirement.md).

---

## 1. Purpose

Provider SPI (Service Provider Interface) là hợp đồng chuẩn cho phép tích hợp nhiều AI Provider vào Adapter mà không ảnh hưởng đến:

- OpenAI API Layer
- Agent Service Layer
- Session Layer
- Cache Layer

Mục tiêu:

```text
Add New Provider
      ↓
No API Changes
No Agent Changes
No Client Changes
```

---

```text
OpenCode
    |
```

OpenAI Layer chỉ làm việc với Provider SPI.

Không được phép gọi trực tiếp Provider Implementation.

---

## 3. Design Principles

## Liskov Substitution

Mọi Provider phải thay thế được cho nhau.

Ví dụ:

```python
provider = GeminiProvider()
```

hoặc

```python
provider = PrivateGPTProvider()
```

Agent Service không cần thay đổi.

---

## Open Closed Principle

Có thể thêm Provider mới.

Không sửa code Agent Service.

---

## Async First

Toàn bộ SPI phải async.

Không hỗ trợ blocking implementation.

---

## Streaming Native

Mọi Provider phải hỗ trợ streaming hoặc cung cấp fallback.

## 5. SPI Interface

```python
from abc import ABC
from abc import abstractmethod

class ProviderAdapter(ABC):

    @abstractmethod
    async def initialize(self):
        pass

    @abstractmethod
    async def health_check(self):
        pass

    @abstractmethod
    async def list_models(self):
        pass

    @abstractmethod
    async def chat_completion(self, request):
        pass

    @abstractmethod
    async def stream_completion(self, request):
        pass

    @abstractmethod
    async def create_conversation(self):
        pass

    @abstractmethod
    async def get_capabilities(self):
        pass
```

## 9. Provider Registration

Provider Registry chịu trách nhiệm đăng ký provider.

```python
class ProviderRegistry:
    register()
    unregister()
    get_provider()
```

Ví dụ:

```python
registry.register(
    "privategpt",
    PrivateGPTProvider()
)
```

## 10. Provider Resolution

Provider được xác định theo model.

Ví dụ:

```text
privategpt/gemini-2.5-pro
```

=>

```python
PrivateGPTProvider
```

Ví dụ:

```text
openai/gpt-5
```

=>

```python
OpenAIProvider
```

## 12. Session SPI

Provider có thể yêu cầu session.

```python
class SessionContext:

    user_id: str
    session_id: str
    conversation_id: str
```

PrivateGPT Provider sử dụng Browser Session.

Các Provider khác có thể dùng API Key.

---

## 13. Error Contract

Provider không trả exception thô.

Provider phải chuyển đổi sang Unified Error.

```python
class ProviderError(Exception):

    provider: str
    error_code: str
    message: str
```

Ví dụ:

```python
ProviderError(
    provider="privategpt",
    error_code="SESSION_EXPIRED",
    message="Session expired"
)
```

---

## 14. Retry Contract

Provider không tự retry.

Retry được quản lý bởi:

```text
Retry Layer
```

Provider chỉ ném exception.

---

## 15. Circuit Breaker Contract

Provider không tự quản lý circuit breaker.

Circuit Breaker nằm ngoài Provider.

```text
Provider
   ↓
Retry Layer
   ↓
Circuit Breaker Layer
```

## 21. Final Rule

Provider SPI là boundary chính của hệ thống.

Mọi backend AI phải đi qua Provider SPI.

Không được phép:

```python
AgentService
    ↓
PrivateGPTClient
```

Bắt buộc:

```python
AgentService
    ↓
Provider SPI
    ↓
Provider Implementation
```
