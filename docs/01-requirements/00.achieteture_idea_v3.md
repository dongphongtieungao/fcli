# OpenCode PrivateGPT Bridge

### Vai trò phụ lục

Đây là **phụ lục kiến trúc đã công bố** của [requirement chính](./00.requirement.md). Nó mô tả rationale, boundary Python và hướng triển khai MVP; không được mở rộng scope, API surface hoặc provider capability ngoài requirement chính. Chi tiết PrivateGPT/OpenCode chưa có runtime capture vẫn là validation condition và phải được xác nhận trước khi implementation được xem là hoàn tất.

### Architecture and Implementation Guide
#### Version 3.0
#### Target audience: Solution Architect, Python Developer, DevOps Engineer, Internal Tooling Team

---

## 1. Tóm tắt điều hành

### 1.1 Định nghĩa sản phẩm

OpenCode PrivateGPT Bridge là một công cụ Python chạy cục bộ trên máy lập trình viên, cung cấp lớp tương thích OpenAI API để OpenCode CLI có thể sử dụng PrivateGPT như backend AI chính.

Ứng dụng không được định vị là AI Gateway enterprise, không phải nền tảng multi tenant, không phải dịch vụ dùng chung quy mô lớn. Bản chất đúng là một developer sidecar adapter chạy trên developer workstation.

Mục tiêu cốt lõi:

1. Tích hợp OpenCode CLI với PrivateGPT mà không sửa OpenCode CLI.
2. Expose API local theo chuẩn tương thích OpenAI.
3. Chuyển đổi request từ OpenCode sang PrivateGPT Internal API.
4. Chuyển đổi response và streaming event từ PrivateGPT về định dạng OpenAI compatible.
5. Quản lý session đăng nhập PrivateGPT lấy từ browser login hoặc cơ chế session nội bộ.
6. Ưu tiên Agent Mode để phục vụ workflow Spec Driven Development.
7. Chạy đơn giản trên máy dev, dễ cấu hình, dễ debug, dễ đóng gói.

### 1.2 Định vị kiến trúc

Tên kiến trúc đề xuất:

**Developer Local Bridge Architecture**

Biến thể cụ thể:

**Modular Monolith plus Embedded FastAPI plus OpenAI Compatibility Adapter plus PrivateGPT Session Bridge**

Ứng dụng gồm một process Python chính:

```text
Developer Workstation
    |
    | uses
    v
OpenCode CLI
    |
    | OpenAI compatible HTTP
    v
OpenCode PrivateGPT Bridge
    |
    | PrivateGPT internal HTTP API with browser/session authentication
    v
PrivateGPT Platform
```

### 1.3 Nguyên tắc thiết kế

1. Local first.
2. Compatibility first.
3. Agent first.
4. Streaming safe.
5. Simple but production disciplined.
6. Không over engineer theo hướng distributed gateway.
7. Không hardcode secret.
8. Không log cookie, token, prompt nhạy cảm.
9. Module hóa rõ ràng để thay đổi PrivateGPT API không phá OpenCode interface.
10. Coi translator layer là phần quan trọng nhất của hệ thống.

## 2. Bối cảnh và vấn đề cần giải quyết

### 2.1 Bối cảnh

Developer muốn dùng OpenCode CLI trong quy trình lập trình theo hướng SDD. OpenCode CLI thường kỳ vọng backend AI có OpenAI compatible API, ví dụ:

```text
GET  /v1/models
POST /v1/chat/completions
POST /v1/responses (không thuộc API surface MVP hiện tại)
```

Trong khi đó PrivateGPT là hệ thống nội bộ, có UI đăng nhập bằng browser, có session, có API nội bộ hoặc API không hoàn toàn tương thích OpenAI.

Do đó cần một bridge local để đứng giữa OpenCode CLI và PrivateGPT.

### 2.2 Vấn đề cốt lõi

Vấn đề không phải là xây một LLM Gateway mạnh.

Vấn đề cốt lõi là làm cho ba phía hiểu nhau:

```text
OpenCode CLI expectation
    |
    | OpenAI API contract
    v
Bridge translation and session handling
    |
    | PrivateGPT internal contract
    v
PrivateGPT actual behavior
```

### 2.3 Ràng buộc

1. App chạy trên máy dev.
2. App là Python console app có embedded web server.
3. Web server expose REST API local.
4. PrivateGPT có thể cần browser login/session reuse.
5. Phải hỗ trợ streaming.
6. OpenCode CLI là consumer chính.
7. Agent Mode là mode chính.
8. Không thiết kế theo hướng service dùng chung cho nhiều team ở giai đoạn đầu.
9. Không cần Kubernetes.
10. Không cần horizontal scaling.
11. Không cần multi provider fallback ở phase đầu.

## 3. Architecture Drivers

### 3.1 Functional Drivers

| Driver | Mô tả | Mức ưu tiên |
|---|---|---|
| OpenAI Compatibility | OpenCode gọi được bridge như OpenAI provider | Critical |
| PrivateGPT Session Reuse | Bridge dùng được session đăng nhập PrivateGPT | Critical |
| Streaming Translation | Stream token/event về OpenCode đúng format | Critical |
| Agent Mode | Hỗ trợ Agent API là luồng chính | Critical |
| CLI Runtime | Start, login, doctor, status, config | High |
| Local Config | Cấu hình base URL, model, port, timeout | High |
| Diagnostics | Kiểm tra bridge, session, PrivateGPT, OpenCode config | High |
| Logging | Log JSON, request id, lỗi dễ debug | High |
| Packaging | Có thể đóng gói thành exe hoặc chạy bằng uv/pipx | Medium |

### 3.2 Non Functional Drivers

| Driver | Định hướng |
|---|---|
| Maintainability | Modular monolith, module boundary rõ |
| Security | Local bind 127.0.0.1, mã hóa session store nếu có thể |
| Reliability | Retry pre stream, graceful error khi session expired |
| Performance | Async HTTP client, streaming non blocking |
| Operability | doctor command, health endpoint, log rõ ràng |
| Simplicity | Không dùng queue, Redis, K8s ở phase đầu |
| Testability | Unit test mapper, integration test mock PrivateGPT |

## 9. Runtime Commands

### 9.1 CLI Commands

```text
bridge start
bridge login
bridge logout
bridge status
bridge doctor
bridge config show
bridge config set privategpt.base_url <url>
bridge config set server.port 4100
```

### 9.2 Command Semantics

#### bridge start

Khởi động embedded FastAPI server tại `127.0.0.1:4100`.

#### bridge login

Mở browser để user đăng nhập PrivateGPT, sau đó lưu session local.

#### bridge status

Hiển thị trạng thái:

1. Bridge server.
2. PrivateGPT base URL.
3. Session valid hay expired.
4. Model mặc định.
5. OpenCode config hint.

#### bridge doctor

Kiểm tra toàn bộ điều kiện chạy:

1. Python version.
2. Port available.
3. Config file exists.
4. Session store readable.
5. PrivateGPT reachable.
6. `/v1/models` local hoạt động.
7. Streaming smoke test nếu có mock endpoint.

```yaml
  enabled: true

logging:
  level: INFO
  format: json
```

## 12. OpenAI Compatible API Surface

### 12.1 GET /v1/models

Mục tiêu: OpenCode biết danh sách model có thể dùng.

Response:

```json
{
  "object": "list",
  "data": [
    {
      "id": "gemini-2.5-pro",
      "object": "model",
      "created": 0,
      "owned_by": "privategpt"
    }
  ]
}
```

### 12.2 POST /v1/chat/completions

Non streaming request:

```json
{
  "model": "gemini-2.5-pro",
  "messages": [
    {
      "role": "user",
      "content": "Explain this repository"
    }
  ],
  "stream": false
}
```

OpenAI compatible response:

```json
{
  "id": "chatcmpl-local-001",
  "object": "chat.completion",
  "created": 0,
  "model": "gemini-2.5-pro",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "..."
      },
      "finish_reason": "stop"
    }
  ]
}
```

### 12.3 POST /v1/chat/completions stream

Request:

```json
{
  "model": "gemini-2.5-pro",
  "messages": [
    {
      "role": "user",
      "content": "Review this code"
    }
  ],
  "stream": true
}
```

SSE response:

```text
data: {"id":"chatcmpl-local-001","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"role":"assistant"},"finish_reason":null}]}

data: {"id":"chatcmpl-local-001","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"content":"Hello"},"finish_reason":null}]}

data: {"id":"chatcmpl-local-001","object":"chat.completion.chunk","choices":[{"index":0,"delta":{},"finish_reason":"stop"}]}

data: [DONE]
```

### 12.4 API Surface ngoài MVP

`/v1/responses`, public `/v1/agents` routes, WebSocket và các API OpenAI-compatible khác không thuộc MVP. Adapter vẫn map chat completion sang PrivateGPT Agent execution nội bộ; điều đó không tạo thêm endpoint public. Bổ sung surface mới cần sửa requirement chính và contract tương ứng.

## 13. API Mapping

### 13.1 Chat Completion Request Mapping

| OpenAI Field | Internal Field | PrivateGPT Field | Ghi chú |
|---|---|---|---|
| model | model_id | model | Map alias nếu cần |
| messages | conversation | prompt/history | Preserve role |
| stream | stream | stream | true/false |
| temperature | generation.temperature | temperature | Optional |
| max_tokens | generation.max_tokens | max_tokens | Optional |
| tools | `[]` | `[]` | Tool payload không được thực thi trong MVP. |
| tool_choice | reject/ignore theo compatibility spike | n/a | Không được bật provider tool policy. |

### 13.2 Message Role Mapping

| OpenAI Role | PrivateGPT Role | Ghi chú |
|---|---|---|
| system | system | Có thể gộp vào instruction |
| user | user | Direct |
| assistant | assistant | History |
| tool | unsupported | Adapter trả lỗi rõ ràng hoặc bỏ qua an toàn theo compatibility spike; không thực thi. |
| developer | system | Nếu OpenCode gửi developer role |

### 13.3 Response Mapping

| PrivateGPT | OpenAI | Ghi chú |
|---|---|---|
| answer | choices[0].message.content | Non stream |
| token | choices[0].delta.content | Stream |
| done | finish_reason=stop | End stream |
| error | OpenAI error object | Standardize |
| agent_event | redacted adapter metadata hoặc error | Không phát sinh OpenAI tool call để thực thi. |

## 14. Session Management Architecture

### 14.1 Session Lifecycle

```text
NO_SESSION
    |
    | bridge login
    v
BROWSER_LOGIN_STARTED
    |
    | user authenticated
    v
SESSION_CAPTURED
    |
    | validate session
    v
SESSION_VALID
    |
    | expiry detected
    v
SESSION_EXPIRED
    |
    | login again or refresh
    v
SESSION_VALID
```

### 14.2 Session Store Contract

```python
class SessionStore:
    def load(self) -> "SessionData | None":
        ...

    def save(self, session: "SessionData") -> None:
        ...

    def clear(self) -> None:
        ...

    def exists(self) -> bool:
        ...
```

### 14.3 Session Data Model

```python
from pydantic import BaseModel
from typing import Optional, Dict

class SessionData(BaseModel):
    base_url: str
    cookies: Dict[str, str]
    headers: Dict[str, str] = {}
    expires_at: Optional[int] = None
    created_at: int
    updated_at: int
```

### 14.4 Session Validation

Bridge phải validate session bằng một endpoint nhẹ của PrivateGPT:

```text
GET /me
GET /api/user
GET /api/session
```

Nếu không có endpoint chính thức, dùng một endpoint an toàn mà UI vẫn gọi sau khi login.

### 14.5 Session Expired Behavior

Khi session expired:

OpenAI compatible error:


### Implementation Options

#### Local Callback Flow

Đây là flow MVP khi PrivateGPT hỗ trợ redirect/callback.

Flow:

1. Bridge mở local callback server.
2. Browser login xong redirect về `http://127.0.0.1:<port>/callback`.
3. Bridge lưu token/cookie.

Refresh/session material được lưu bằng secure store đã được phê duyệt; access token chỉ cache trong memory. Manual cookie import và browser-profile reading không thuộc MVP vì không đáp ứng acceptance login không cần copy token và làm tăng rủi ro credential handling. Nếu callback không được PrivateGPT hỗ trợ, cần quyết định security riêng trước khi triển khai một alternative flow.

## 20. Retry Policy

### 20.1 Retry Allowed

| Lỗi | Retry | Điều kiện |
|---|---|---|
| Connection refused | Yes | Chưa gửi token |
| Connect timeout | Yes | Chưa gửi token |
| Read timeout before first token | Yes | Chưa gửi token |
| HTTP 429 | Yes | Chưa gửi token |
| HTTP 502 | Yes | Chưa gửi token |
| HTTP 503 | Yes | Chưa gửi token |
| HTTP 504 | Yes | Chưa gửi token |

### 20.2 Retry Not Allowed

| Tình huống | Lý do |
|---|---|
| Đã gửi token | Tránh duplicate output |
| User cancel | Không phải lỗi backend |
| Session expired | Cần login lại |
| Bad request | Request sai, retry vô ích |
| Unsupported model | Config sai |

### 20.3 Retry Config

```yaml
retry:
  max_attempts: 2
  base_delay_ms: 500
  max_delay_ms: 3000
  jitter: true
```

## 21. Circuit Breaker

### 21.1 Mục tiêu

Circuit breaker trong bridge không phải để xử lý high scale. Mục tiêu là bảo vệ PrivateGPT và tránh dev bị treo khi backend lỗi liên tục.

### 21.2 Rule đơn giản

```text
If 5 consecutive failures:
    state = OPEN for 30 seconds

After 30 seconds:
    state = HALF_OPEN

If next request succeeds:
    state = CLOSED

If next request fails:
    state = OPEN again
```

### 21.3 Circuit Breaker States

```text
CLOSED
    |
    | too many failures
    v
OPEN
    |
    | cooldown elapsed
    v
HALF_OPEN
    |
    +-- success -> CLOSED
    |
    +-- failure -> OPEN
```

## 23. Backpressure

### 23.1 Mục tiêu

Bridge chạy local, tải thấp. Tuy nhiên streaming vẫn cần bounded buffer để tránh memory leak nếu client chậm hoặc PrivateGPT trả event nhanh.

### 23.2 Rule

1. Không dùng unbounded queue.
2. Buffer mặc định 50 event.
3. Nếu buffer đầy, ưu tiên block producer.
4. Không drop token trừ khi có cấu hình rõ.

### 23.3 Strategy

```text
PrivateGPT stream
    |
    v
Bounded queue size 50
    |
    v
SSE writer
    |
    v
OpenCode CLI
```

## 26. Security Design

### 26.1 Local Bind

Default:

```text
127.0.0.1:4100
```

Không bind `0.0.0.0` mặc định.

### 26.2 API Key Handling

Nếu cần bảo vệ local endpoint:

```yaml

  require_api_key: true
  api_key_env: BRIDGE_API_KEY
```

### 26.3 Session Storage

Ưu tiên mã hóa at rest.

Windows:

1. Dùng Windows Credential Manager nếu có thể.
2. Hoặc dùng keyring library.
3. Nếu file JSON, phải chmod hoặc ACL hạn chế user hiện tại.

### 26.4 Threat Model Tối Thiểu

| Threat | Mitigation |
|---|---|
| Local malware đọc session | Secure store, file permission |
| Expose port ra mạng | Bind 127.0.0.1 |
| Log leak prompt/token | Mask sensitive |
| Session expired | Explicit error |
| CSRF local request | API key optional |
| Cookie reuse sai domain | Validate session base_url |

## 32. Reference Implementation Skeleton

### 32.1 FastAPI App Factory

```python
from fastapi import FastAPI
from app.api.routes import health, models, chat_completions

def create_app() -> FastAPI:
    app = FastAPI(title="OpenCode PrivateGPT Bridge")
    app.include_router(health.router)
    app.include_router(models.router, prefix="/v1")
    app.include_router(chat_completions.router, prefix="/v1")
    return app
```

### 32.2 Chat Completion Route

```python
from fastapi import APIRouter
from fastapi.responses import StreamingResponse
from app.openai_contract.chat import ChatCompletionRequest
from app.services.chat_service import ChatService

router = APIRouter()

@router.post("/chat/completions")
async def chat_completions(request: ChatCompletionRequest):
    service = ChatService()

    if request.stream:
        async def event_generator():
            async for event in service.stream_chat(request):
                yield event

        return StreamingResponse(
            event_generator(),
            media_type="text/event-stream"
        )

    return await service.chat(request)
```

### 32.3 Stream Mapper

```python
import json

def openai_token_chunk(request_id: str, model: str, token: str) -> str:
    payload = {
        "id": request_id,
        "object": "chat.completion.chunk",
        "created": 0,
        "model": model,
        "choices": [
            {
                "index": 0,
                "delta": {
                    "content": token
                },
                "finish_reason": None
            }
        ]
    }
    return f"data: {json.dumps(payload, ensure_ascii=False)}\n\n"

def openai_done_chunk() -> str:
    return "data: [DONE]\n\n"
```

## 35. Final Architecture Decision

OpenCode PrivateGPT Bridge sẽ được triển khai theo kiến trúc Modular Monolith chạy local trên developer workstation.

Quyết định chính:

1. FastAPI embedded server làm OpenAI compatible API endpoint.
2. CLI runtime quản lý lifecycle, login, status, doctor.
3. Translator layer là module trung tâm.
4. PrivateGPT integration được cô lập trong adapter riêng.
5. Session management là capability first class.
6. Agent Mode là execution path ưu tiên.
7. Streaming phải cancel safe và retry safe.
8. Retry chỉ xảy ra trước khi stream bắt đầu.
9. Circuit breaker đơn giản để tránh spam PrivateGPT khi lỗi.
10. Không dùng distributed architecture ở phase đầu.

Kiến trúc này phù hợp vì nó giải quyết đúng bài toán: giúp OpenCode CLI dùng được PrivateGPT trong môi trường lập trình viên, phục vụ SDD workflow, không over engineer, nhưng vẫn đủ kỷ luật để dev triển khai production grade local tool.
