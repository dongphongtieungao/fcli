# Vision

## Purpose

Define a reusable vision framework for aligning product direction, technical direction, constraints, and non-goals.

## Context

This file should remain useful before the new project has a product domain. It defines the questions and decision shape that future project vision work should fill.

## Vision Elements

| Element | Reusable Question | Evidence |
|---|---|---|
| Problem | What problem should the project solve? | Requirement, user decision, discovery note |
| Users | Who benefits from the project or operates it? | Use case, stakeholder map |
| Outcome | What measurable outcome should change? | Acceptance criteria, KPI, QA signal |
| Scope | What is inside and outside the first useful release? | Requirement and DoR |
| Constraints | What limits architecture, data, compliance, cost, or operation? | NFR, ADR, governance |
| Non-goals | What should the project explicitly not try to solve? | Decision log |

## Vision Statement Shape

```text
For <user or stakeholder>,
the project provides <capability>
so that <outcome>.
It is constrained by <important limits>
and it intentionally does not cover <non-goals>.
```

## Promotion Rules

Promote vision updates when user decisions clarify goals, non-goals, scope, or success criteria. Link those updates to requirements, C4 architecture, NFRs, and ADRs when they influence design or implementation.

## Starter Position

The project-specific vision is not defined yet. The reusable vision framework above is active.
