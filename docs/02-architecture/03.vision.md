# Tầm nhìn sản phẩm — OpenCode PrivateGPT Bridge MVP

## Mục đích

Định hướng sản phẩm, kiến trúc và tiêu chí thành công cho OpenCode PrivateGPT Bridge MVP. Tài liệu này giúp các quyết định thiết kế và triển khai luôn phục vụ cùng một kết quả: đưa năng lực PrivateGPT nội bộ vào quy trình coding trên terminal bằng OpenCode, nhưng không biến adapter thành một coding agent thứ hai.

## Bộ requirement đã công bố

[Requirement chính](../01-requirements/00.requirement.md) là điểm vào duy nhất cho scope MVP. Các phụ lục dưới đây cụ thể hóa nó theo trách nhiệm riêng; Vision chỉ tổng hợp định hướng, không thay thế contract hoặc acceptance criteria của chúng.

Khi có khác biệt, requirement chính có ưu tiên cao nhất. Chi tiết PrivateGPT/OpenCode chưa có capture runtime là validation condition, không phải capability đã được xác minh.

## Bối cảnh và vấn đề

PrivateGPT nội bộ đã có API và cơ chế Agent, nhưng không cung cấp trực tiếp giao diện OpenAI-compatible mà OpenCode cần. Plugin IntelliJ cũ chứng minh hướng tích hợp có thể thực hiện, song không phù hợp làm workflow chính vì phụ thuộc IntelliJ runtime và có logic tool loop riêng.

Developer cần một cách dùng PrivateGPT ổn định, độc lập với IntelliJ, trong source repository và terminal. Nếu không có adapter, họ bị khóa vào workflow plugin, khó phân biệt lỗi client/API/model, và khó chuẩn hóa trải nghiệm AI coding cho nhiều developer.

## Tuyên bố tầm nhìn

Đối với developer sử dụng OpenCode trong source repository, OpenCode PrivateGPT Bridge MVP cung cấp một dịch vụ local chạy trên localhost, chuyển đổi giao tiếp OpenAI-compatible của OpenCode thành các lời gọi PrivateGPT Agent và Conversation API. Nhờ đó developer có thể dùng Gemini 2.5 Pro qua PrivateGPT trong một workflow coding có kiểm soát quyền, trong khi OpenCode vẫn là nơi duy nhất đọc/sửa file, chạy lệnh và yêu cầu phê duyệt.

## Người dùng và giá trị mang lại

| Đối tượng | Nhu cầu | Giá trị nhận được |
|---|---|---|
| Developer / người dùng OpenCode | Dùng PrivateGPT trong terminal và repository mà không cần IntelliJ plugin. | Một provider local cho OpenCode, browser login, hỗ trợ phân tích repository và đề xuất thay đổi có phê duyệt. |
| Đội ngũ phát triển adapter | Một boundary rõ giữa OpenCode, adapter và PrivateGPT. | Có requirement, Provider SPI và kiến trúc để triển khai/kiểm thử mà không sao chép plugin runtime cũ. |
| Đơn vị vận hành và bảo mật | Hạn chế rủi ro token, prompt và source code khi tích hợp AI nội bộ. | Adapter bind localhost, token được bảo vệ, log phải redaction, và không tự thực thi công cụ. |

## Kết quả sản phẩm mong muốn

1. Developer có thể đăng nhập PrivateGPT bằng browser và kiểm tra trạng thái đăng nhập từ adapter.
2. OpenCode có thể chọn model `privategpt/gemini-2.5-pro` và gửi yêu cầu qua endpoint local của adapter.
3. Adapter tạo hoặc đồng bộ PrivateGPT Agent tương thích OpenCode trước khi gửi hội thoại.
4. Mỗi yêu cầu hội thoại có `metadata.agent_id`, được chuyển tới PrivateGPT Conversation API và trả về theo định dạng OpenAI-compatible.
5. Developer hoàn thành một tác vụ phân tích repository không phá hủy và nhận được đề xuất sửa đổi nhỏ theo cơ chế phê duyệt của OpenCode.

## Phi mục tiêu

MVP chủ động không giải quyết các nội dung sau:

- Thay thế OpenCode bằng một agent runtime trong adapter.
- Phụ thuộc bắt buộc vào LiteLLM, triển khai gateway tập trung hoặc hỗ trợ nhiều client chính thức.
- Hỗ trợ model khác, multi-model routing hoặc fallback tự động.
- Port IntelliJ runtime, `AgentLoopRunner`, `ToolRegistry` hoặc plugin-style `<tool_call>` sang adapter.
- Tự đọc/sửa file, tự chạy command, quản lý quyền thay OpenCode, hoặc lưu toàn bộ lịch sử hội thoại.
- Bổ sung image input, file-upload UI, browser UI automation hoặc multi-user deployment trong MVP.

## Nguyên tắc định hướng kiến trúc

1. **OpenCode làm chủ workflow:** mọi hành động lên workspace, file, command và approval thuộc OpenCode.
2. **Adapter là boundary tích hợp:** adapter che giấu auth và API nội bộ PrivateGPT, đồng thời chỉ thực hiện mapping, sync Agent và response parsing.
3. **Agent API là bắt buộc:** instruction được quản lý trong PrivateGPT Agent; không fallback âm thầm sang instruction inline.
4. **Giới hạn biến số trong MVP:** chỉ một model công khai, một client chính và một path runtime local.
5. **An toàn theo mặc định:** bind localhost, không log token/raw source code, không thay thế model và không thực thi tool call.
6. **Mở rộng có kiểm soát:** Provider SPI tách concrete provider khỏi OpenAI API layer và Agent Service layer để cho phép mở rộng sau MVP.

## Ràng buộc thành công

| Nhóm ràng buộc | Ràng buộc |
|---|---|
| Runtime | Adapter chạy local tại `127.0.0.1` theo mặc định; implementation chính là Python. |
| Client | OpenCode CLI là coding client chính duy nhất của MVP. |
| Model | Chỉ `gemini-2.5-pro` được expose; không fallback sang model khác. |
| Tích hợp | Conversation phải đi qua PrivateGPT Agent đã đồng bộ và mang `metadata.agent_id`. |
| Bảo mật | Không hardcode hoặc log token, cookie, password hay raw source code; token không được lưu plaintext. |
| Vận hành | Adapter cần cung cấp lệnh login, status, agent sync/status/reset và doctor để chẩn đoán. |

## Tín hiệu thành công và bằng chứng

| Tín hiệu | Bằng chứng tối thiểu |
|---|---|
| Sẵn sàng xác thực | Browser login thành công, token được lưu an toàn, refresh được kiểm chứng và log được redaction. |
| Sẵn sàng model/Agent | `/v1/models` chỉ trả Gemini 2.5 Pro; Agent API tạo/cập nhật được Agent đúng instruction/model. |
| Tương thích hội thoại | Direct HTTP smoke test và OpenCode request capture xác nhận request/response mapping cùng SSE parser. |
| An toàn workflow | OpenCode yêu cầu phê duyệt trước edit/command; adapter không có tool executor. |
| Giá trị pilot | OpenCode thực hiện phân tích repository không phá hủy và đề xuất một sửa đổi nhỏ có phê duyệt. |

## Điều kiện cần xác minh trước pilot

Tầm nhìn này không thay thế các bằng chứng runtime còn thiếu. Trước khi khẳng định MVP đạt mục tiêu, cần xác minh model ID nội bộ của Gemini 2.5 Pro, response thực tế của Model/Agent/Conversation API, request shape do OpenCode gửi, yêu cầu streaming/native tool call, quyết định token store và chính sách gửi source code nội bộ tới PrivateGPT.
