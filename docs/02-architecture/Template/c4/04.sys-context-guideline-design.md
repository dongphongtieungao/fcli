# C4 System Context Guideline

## Purpose

Guide creation of C4 system context, container, and component diagrams.

## Context

Use this as a neutral checklist. Put generated diagrams under the project diagrams folder.

## Checklist

- [ ] Actors are named.
- [ ] External systems are named.
- [ ] Trust boundaries are clear.
- [ ] Data flows are clear.
- [ ] Diagram source is committed when possible.
