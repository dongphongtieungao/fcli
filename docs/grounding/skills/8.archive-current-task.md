# Archive Current Task Skill

## Purpose

Archive `llm-wiki/tasks/current/` into `llm-wiki/tasks/archive/` and leave `current` clean. A populated fresh `current` workspace for the next task should be created only by Skill 9 from the next task description.

This skill preserves task memory without promoting workspace notes into official `docs/`.

## Context

`llm-wiki/` is the controlled agent workspace. Task folders under `llm-wiki/tasks/` are workspace memory only. They must not override `docs/`, executable specs, contracts, ADRs, QA matrix, runbooks, or current user instructions.

## When to use

Use this skill:

1. Before starting a new task when `llm-wiki/tasks/current/` has useful content.
2. After finishing a task that produced workspace notes, decisions, verification notes, or promotion records.
3. When the user asks to archive the current task workspace.

Do not use this skill to archive official docs or source code.

## Folder convention

Current workspace:

```text
llm-wiki/tasks/current/
```

Archive target:

```text
llm-wiki/tasks/archive/YYYYMMDD_NNN_short-name/
```

Examples:

```text
llm-wiki/tasks/archive/20250912_005_adjust-threshold/
llm-wiki/tasks/archive/20260513_001_task-workspace-setup/
```

Use zero-padded sequence numbers (`001`, `002`, `010`) so folders sort correctly.

`llm-wiki/tasks/archive/` is historical workspace memory. It is searched only when a task needs history lookup; it is not part of the default always-scan working set.

## Inputs

1. Current date in `YYYYMMDD`.
2. Short task name in lowercase kebab-case.
3. Next sequence number for the date.
4. Current folder contents.
5. Optional next task description when archive is immediately followed by new task creation.

## Procedure

### Step 1. Inspect `current`

Check whether `llm-wiki/tasks/current/` exists and has meaningful content.

Meaningful content includes any file beyond a fresh placeholder, or any edited task brief, raw input, intake, decision log, promotion note, analysis, or verification note.

### Step 2. Choose archive folder

Find existing folders matching:

```text
llm-wiki/tasks/archive/YYYYMMDD_*
```

Choose the next zero-padded sequence number for that date.

If the user supplied a short name, use it. Otherwise derive a conservative short name from `TASK.md` title or request summary. If no good name exists, use `untitled-task`.

### Step 3. Move current content

Move the contents of:

```text
llm-wiki/tasks/current/
```

to:

```text
llm-wiki/tasks/archive/YYYYMMDD_NNN_short-name/
```

Preserve filenames, links, frontmatter, and original meaning.

### Step 4. Prepare for a fresh `current`

After archive, clean `current` by deleting its remaining files and subdirectories, then recreate only the empty directory:

```text
llm-wiki/tasks/current/
```

Do not copy blank template files. Task workspace methodology lives in `docs/methodology/Task.md`.

If a next task description is available, call Skill 9 to create task-specific `current` files from that description. Do not leave template files in `current` during archive-only completion.

### Step 5. Report archive result

Report:

```text
Archive Report
Archived from:
Archived to:
New current workspace:
Files moved:
Fresh current directory:
Next task skill:
Promotion needed:
Docs changed:
Remaining risk:
```

## Safety rules

1. Do not delete archived task folders.
2. Do not overwrite an existing archive folder.
3. Do not promote workspace content to `docs/` during archive unless explicitly requested.
4. Do not change official docs, manifest, sources, or prompts during ordinary task archive.
5. Do not use destructive shell pipelines. Move files with path checks.
6. If `current` is empty or only contains a fresh placeholder, report that no archive was needed.
7. Do not create a new populated `current` without a task description; use Skill 9 for that.

## Output contract

```text
Archive Current Task Report
Current task had content:
Archive folder:
Sequence selected:
Files archived:
Fresh current directory:
Next task skill:
Docs promotion performed:
Manifest or sources update needed:
Final status:
```

## Done criteria

1. `llm-wiki/tasks/current/` exists as an empty directory or has been recreated by Skill 9.
2. `docs/methodology/Task.md` exists.
3. If current content was meaningful, it exists under `llm-wiki/tasks/archive/YYYYMMDD_NNN_short-name/`.
4. No official source of truth was changed by archive alone.
5. Archive target does not collide with existing task history.
6. If a new task starts immediately, Skill 9 created task-specific current files.
