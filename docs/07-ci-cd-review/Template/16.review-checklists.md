# Review Checklist Template

## Purpose

Give reviewers a repeatable checklist for project changes.

## Context

Tailor this after the new project defines modules, risks, and quality gates.

## Checklist

- [ ] Scope is clear.
- [ ] Tests or verification evidence are present.
- [ ] No secrets or sensitive data were introduced.
- [ ] Documentation impact was considered.
