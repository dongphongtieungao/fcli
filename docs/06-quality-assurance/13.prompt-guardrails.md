# Prompt Guardrails

## Purpose

Define reusable guardrails for agent prompts and outputs.

## Context

The new project has no domain-specific prompt rules yet. These guardrails apply to starter agent work.

## Guardrails

| Guardrail | Rule |
|---|---|
| Grounding | Read relevant official docs before durable claims. |
| Safety | Do not expose secrets, credentials, tokens, or sensitive data. |
| Verification | Do not claim checks passed unless they were run. |
| Scope | Keep changes within the requested module or document set. |
| Drift | Report when docs, tests, and source disagree. |

## Output Contract

| Output Area | Required Behavior |
|---|---|
| Sources | Name official docs, specs, tests, or task notes actually used. |
| Assumptions | Separate assumptions from confirmed facts. |
| Evidence | Report command, exit code, artifact, warning, or reason not run. |
| Promotion | State whether durable findings need docs/spec/runbook sync. |
| Safety | Exclude secrets, credentials, PII, and sensitive raw data. |

## Maintenance Rules

Promote a prompt guardrail when it prevents repeated agent mistakes or controls task safety across multiple work types. Keep task-specific prompt advice in task notes until it proves reusable.
