# KMS Integrate: Lightweight KMS bằng rg, Grounding Manifest và Obsidian Metadata

## Purpose

Tài liệu này hướng dẫn triển khai và vận hành Lightweight KMS cho project đã chuẩn hóa tài liệu theo Obsidian compatible Markdown.

Trong [knowledge_base.md](./knowledge_base.md), tài liệu này là nguồn chính cho lớp KMS Operations thuộc trụ cột "KMS cho Agentic Development" và liên quan trực tiếp tới "Grounding, Retrieval & Traceability với `AGENTS.md`, manifest, `kb.ps1`, `rg`".

Thiết kế hiện tại dùng:

1. `docs/` làm KMS chính thức.
2. `docs/grounding/manifest.md` làm single human readable grounding map.
3. `docs/grounding/manifest.yaml` làm machine readable source registry.
4. `docs/grounding/sources.md` làm source ID catalog.
5. `docs/grounding/prompts.md` làm prompt governance và prompt routing.
6. `llm-wiki/` làm controlled workspace cho agent.
7. `rg` và PowerShell làm retrieval, audit và traceability layer.
8. `AGENTS.md` làm execution policy cho agent.

Tài liệu này không thay thế `docs/grounding/manifest.md`. `manifest.md` là bản đồ nguồn chính thức. Tài liệu này là runbook vận hành KMS bằng `rg`, Obsidian metadata và agent workflow.

## 3. Grounding precedence

Agent phải xử lý theo thứ tự ưu tiên sau:

```text
1. User task hiện tại
2. AGENTS.md
3. docs/grounding/manifest.md
4. docs/grounding/manifest.yaml nếu cần registry máy đọc
5. docs/grounding/sources.md nếu cần source ID chi tiết
6. docs/grounding/prompts.md nếu cần prompt lane
7. docs/ official artifacts
8. tests/features hoặc tests/bdd
9. llm-wiki workspace context
10. src current implementation
11. agent inference
```

Quy tắc:

1. `docs/` nói hệ thống phải như thế nào.
2. `tests/features/` nói hành vi acceptance nào đã được biểu diễn thành executable spec.
3. `llm-wiki/` nói agent đã quan sát, nghi ngờ hoặc học được gì.
4. `src/` nói implementation hiện tại đang làm gì.
5. Nếu `docs/`, `tests/`, `llm-wiki/` và `src/` mâu thuẫn, agent ghi vào `llm-wiki/contradiction-log.md` trước khi đề xuất sửa.

## 7. Quy trình retrieval bằng rg

### 7.1. Kiểm tra grounding entrypoint

```powershell
rg -n "Grounding|Coverage|Source|Manifest|Current Grounding|Maintenance Rules" docs/grounding/manifest.md
rg -n "source_id|path_or_url|role|coverage|owner" docs/grounding/manifest.yaml
rg -n "Source ID|Path|Owner|Coverage" docs/grounding/sources.md
rg -n "Prompt|lane|grounding|verify|sync docs" docs/grounding/prompts.md
```

### 7.2. Tìm theo metadata

```powershell
rg -n -i "type:|category:|status:|truth_level:|source_policy:|tags:" docs llm-wiki -g "*.md"
```

### 7.3. Tìm official docs theo task

```powershell
rg -n -i "<task keyword>|<synonym>|<module>|<feature>" docs tests
```

### 7.4. Tìm workspace context

```powershell
rg -n -i "<task keyword>|<synonym>|<module>|<feature>" llm-wiki
```

### 7.5. Tìm implementation hiện tại

```powershell
rg -n -i "<task keyword>|<class>|<function>|<api>|<module>" src tests
```

### 7.6. Tìm blocker trước khi code

```powershell
rg -n -i "BLOCKED|TBD|TODO|FIXME|unclear|assumption|open question|contradiction" docs llm-wiki tests
```

## 8. Agent workflow chuẩn

```text
1. Đọc AGENTS.md.
2. Đọc docs/grounding/manifest.md.
3. Nếu cần source ID, đọc manifest.yaml và sources.md.
4. Nếu cần prompt lane, đọc prompts.md.
5. Dùng rg tìm official artifacts trong docs/.
6. Dùng rg tìm BDD hoặc tests liên quan.
7. Đọc llm-wiki để xem workspace context.
8. Đọc src để verify hiện trạng implementation.
9. Đưa ra task brief.
10. Nếu READY, lập plan và implement.
11. Verify bằng tests, contract, checklist hoặc manual inspection.
12. Update llm-wiki nếu có tri thức mới.
13. Update docs chỉ khi source of truth chính thức thay đổi.
14. Nếu artifact mới được thêm, cập nhật manifest.md, manifest.yaml và sources.md.
```

## 8.8. Runbook triển khai template và taxonomy suggestion trong Obsidian

### 8.8.1. Mục tiêu

Runbook này giúp cấu hình sẵn template và taxonomy để khi dùng Obsidian, các trường như `category`, `type`, `status`, `truth_level`, `source_policy` và `tags` có giá trị mẫu thống nhất. Mục tiêu là giúp người viết tài liệu chọn đúng danh mục, giảm typo và giúp agent tìm kiếm bằng `rg` ổn định hơn.

Lưu ý: Obsidian core hỗ trợ Properties, Templates và Tags, nhưng không khóa enum cứng như database constraint. Vì vậy cần kết hợp 4 lớp:

1. Template chuẩn.
2. Taxonomy document.
3. Seed notes chứa giá trị mẫu để Obsidian dễ gợi ý.
4. Audit bằng `rg` hoặc PowerShell để phát hiện sai value.

### 8.8.2. Cấu trúc thư mục cần tạo

```text
docs/

    taxonomy.md

  _templates/

    llm-wiki-template.md

      category/

      tags/
```

### 8.8.3. Cấu hình Obsidian

Thực hiện trong Obsidian:

1. Mở Settings.
2. Vào Core plugins.
3. Bật Templates.
4. Bật Properties nếu phiên bản Obsidian hỗ trợ Properties.
5. Bật Tags view nếu muốn quản lý tag trực quan.
6. Vào Templates.
7. Đặt Template folder location là:

```text
docs/_templates
```

Sau đó tạo note mới và dùng lệnh:

```text
Insert template
```

để chèn metadata chuẩn.

### 8.8.10. PowerShell tạo seed taxonomy tự động

Tạo file:

```text
tools/init-obsidian-taxonomy.ps1
```

Nội dung:

```powershell
$ErrorActionPreference = "Stop"

$base = "docs/_obsidian/property-values"

$values = @{
  "category" = @(
    "grounding", "governance", "methodology", "requirement", "architecture",
    "business_analysis", "spec_test", "quality_assurance", "ci_cd_review",
    "runbook", "operation", "prompt_governance", "agent_workspace",
    "workspace_index", "workspace_memory", "workspace_source_map",
    "workspace_codebase", "workspace_module", "workspace_question",
    "workspace_assumption", "workspace_contradiction", "workspace_risk",
    "workspace_lesson", "workspace_verification"
  )
  "type" = @(
    "methodology", "guide", "technical_note", "governance", "manifest",
    "source_catalog", "prompt_governance", "vision", "glossary", "requirement",
    "use_case", "architecture_c4", "nfr", "adr", "standards", "bpmn",
    "executable_spec", "openapi", "application_spec", "ui_spec", "prompt_guardrails",
    "qa_matrix", "quality_matrix", "review_checklist", "dor", "dod", "runbook",
    "workspace_index", "source_map", "project_memory", "codebase_map", "module_note",
    "task_memory", "open_questions", "assumptions", "contradiction_log", "risk_log",
    "lessons_learned", "verification_notes", "agent_runbook"
  )
  "status" = @(
    "draft", "reviewed", "published", "deprecated", "archived",
    "observed", "synthesized", "verified", "promoted", "stale", "discarded"
  )
  "truth_level" = @(
    "official", "working", "observed", "assumption", "archived"
  )
  "source_policy" = @(
    "source_of_truth", "source_catalog", "machine_registry", "methodology",
    "prompt_governance", "workspace_only", "reference_only"
  )
}

foreach ($group in $values.Keys) {
  $dir = Join-Path $base $group
  New-Item -ItemType Directory -Force -Path $dir | Out-Null

  foreach ($value in $values[$group]) {
    $file = Join-Path $dir "$value.md"
    $content = @"
---
$group: $value
type: technical_note
category: governance
status: published
tags:
  - kms/taxonomy
---

# $group: $value

Seed note for Obsidian property value suggestion.
"@
    Set-Content -Path $file -Value $content -Encoding UTF8
  }
}

Write-Host "Obsidian taxonomy seed notes created under $base"
```

## Phụ lục B. Wrapper `tools/kb.ps1` đầy đủ

Tạo file tùy chọn:

```text
tools/kb.ps1
```

Mục tiêu của wrapper là gom các lệnh `rg` thành command ổn định cho agent và human trên Windows PowerShell.

```powershell
param(
    [Parameter(Position=0)]
    [string]$Command,

    [Parameter(Position=1)]
    [string]$Query
)

$ErrorActionPreference = "Stop"

$Docs = "docs"
$Tests = "tests"
$Src = "src"
$Wiki = "llm-wiki"
$ManifestMd = "docs/grounding/manifest.md"
$ManifestYaml = "docs/grounding/manifest.yaml"
$SourcesMd = "docs/grounding/sources.md"
$PromptsMd = "docs/grounding/prompts.md"

function Require-Rg {
    if (-not (Get-Command rg -ErrorAction SilentlyContinue)) {
        Write-Host "rg is not installed or not available in PATH."
        Write-Host "Install with Chocolatey: choco install ripgrep"
        exit 1
    }
}

function EntryPoints {
    Require-Rg
    Write-Host "Grounding entrypoint"
    rg -n "Grounding|Coverage|Source|Manifest|Maintenance Rules|Current Grounding" $ManifestMd

    Write-Host "Machine registry"
    rg -n "source_id|path_or_url|role|coverage|owner" $ManifestYaml

    Write-Host "Source catalog"
    rg -n "Source ID|Path|Owner|Coverage" $SourcesMd

    Write-Host "Prompt routing"
    rg -n "Prompt|lane|grounding|verify|sync docs" $PromptsMd
}

function Search-All {
    Require-Rg
    if ([string]::IsNullOrWhiteSpace($Query)) {
        Write-Host "Missing query"
        exit 1
    }

    rg -n -i $Query $Docs $Tests $Src $Wiki
}

function Search-Docs {
    Require-Rg
    if ([string]::IsNullOrWhiteSpace($Query)) {
        Write-Host "Missing query"
        exit 1
    }

    rg -n -i $Query $Docs
}

function Search-Wiki {
    Require-Rg
    if ([string]::IsNullOrWhiteSpace($Query)) {
        Write-Host "Missing query"
        exit 1
    }

    rg -n -i $Query $Wiki
}

function Snapshot {
    Require-Rg
    Write-Host "Official foundation artifacts"
    rg -n -i "type: vision|type: glossary|type: architecture_c4|type: nfr|type: adr|type: standards" $Docs -g "*.md"

    Write-Host "Business flow, executable spec, contract, QA"
    rg -n -i "type: bpmn|type: executable_spec|type: openapi|type: application_spec|type: ui_spec|type: prompt_guardrails|type: qa_matrix" $Docs $Tests -g "*.md" -g "*.feature" -g "*.yaml" -g "*.yml"

    Write-Host "Workspace context"
    if (Test-Path $Wiki) {
        rg -n -i "workspace_index|source_map|codebase_map|open_questions|contradiction_log|lessons_learned" $Wiki -g "*.md"
    }

    Write-Host "Blocker markers"
    rg -n -i "BLOCKED|TBD|TODO|FIXME|unclear|assumption|open question|contradiction" $Docs $Tests $Wiki
}

function Ready {
    Require-Rg
    Write-Host "Checking required implementation artifacts"

    rg -n -i "type: vision|03.vision|# .*Vision|scope" $Docs
    rg -n -i "type: glossary|02.glossary|# .*Glossary|domain term|bounded context" $Docs
    rg -n -i "type: standards|08.standards|# .*Standards|coding standard" $Docs
    rg -n -i "type: bpmn|09.bpmn|BPMN|happy path|error path|business flow" $Docs
    rg -n -i "Feature:|Scenario:|Given |When |Then |acceptance criteria" $Docs $Tests
    rg -n -i "openapi:|type: openapi|type: application_spec|type: ui_spec|contract|interface spec" $Docs
    rg -n -i "type: prompt_guardrails|type: qa_matrix|type: review_checklist|type: dor|type: dod|guardrails|QA matrix|review checklist|Definition of Ready|Definition of Done" $Docs
    rg -n -i "BLOCKED|TBD|TODO|FIXME|unclear|assumption|open question|contradiction" $Docs $Wiki $Tests
}

function Audit {
    Require-Rg
    Write-Host "Audit docs and workspace markers"
    rg -n -i "TBD|TODO|FIXME|unclear|assumption|blocked|deprecated|outdated" $Docs $Wiki

    Write-Host "Draft or observed documents"
    rg -n "status: draft|status: observed|status: stale" $Docs $Wiki

    Write-Host "Source of truth references"
    rg -n "source_of_truth:" $Docs $Wiki -g "*.md"

    Write-Host "Metadata fields"
    rg -n "type:|category:|truth_level:|source_policy:|tags:" $Docs $Wiki -g "*.md"

    Write-Host "Related and references"
    rg -n "related:|## Related|## References|verified_against:" $Docs $Wiki -g "*.md"
}

function Missing-Meta {
    Get-ChildItem $Docs,$Wiki -Recurse -Filter *.md -ErrorAction SilentlyContinue | Where-Object {
        -not (Select-String -Path $_.FullName -Pattern "^---" -Quiet)
    } | Select-Object FullName
}

function Missing-H1 {
    Get-ChildItem $Docs,$Wiki -Recurse -Filter *.md -ErrorAction SilentlyContinue | Where-Object {
        -not (Select-String -Path $_.FullName -Pattern "^# " -Quiet)
    } | Select-Object FullName
}

function Trace {
    Require-Rg
    if ([string]::IsNullOrWhiteSpace($Query)) {
        Write-Host "Missing feature query"
        exit 1
    }

    Write-Host "Feature search"
    rg -n -i $Query $Docs $Tests $Src $Wiki

    Write-Host "Trace IDs"
    rg -n -i "REQ-|FLOW-|BDD-|API-|application-|QA-|DOR-|DOD-|ADR-" $Docs $Tests $Src $Wiki

    Write-Host "Trace metadata"
    rg -n -i "source_of_truth:|related:|upstream:|downstream:|## Related|## References" $Docs $Wiki -g "*.md"
}

function Usage {
    Write-Host "Usage:"
    Write-Host ".\tools\kb.ps1 entrypoints"
    Write-Host ".\tools\kb.ps1 snapshot"
    Write-Host ".\tools\kb.ps1 search \"keyword\""
    Write-Host ".\tools\kb.ps1 docs \"keyword\""
    Write-Host ".\tools\kb.ps1 wiki \"keyword\""
    Write-Host ".\tools\kb.ps1 ready"
    Write-Host ".\tools\kb.ps1 audit"
    Write-Host ".\tools\kb.ps1 missing-meta"
    Write-Host ".\tools\kb.ps1 missing-h1"
    Write-Host ".\tools\kb.ps1 trace \"feature\""
}

switch ($Command) {

    "ready" { Ready }
```
