Yêu cầu bắt buộc cần thêm vào AGENTS.md:

1. Obsidian Wiki Documentation Gate

Thêm rule:
Trước khi tạo mới hoặc sửa bất kỳ file .md nào, agent phải xác định:
1. File thuộc loại tài liệu nào.
2. File thuộc artifact nào trong mô hình Spec First.
3. File phụ thuộc source of truth nào.
4. File nên link tới tài liệu nào.
5. File đang là draft, reviewed, published hay deprecated.
6. File có phải là một mắt xích trong chuỗi Vision, Glossary, C4, NFR, ADR, Standards, BPMN, BDD, Contract, QA, DoR, DoD hay không.

Bắt buộc:
1. Không được tạo hoặc sửa file .md theo kiểu isolated note.
2. Không được sửa nội dung làm mất traceability với artifact trước hoặc sau.
3. Không được thêm dữ kiện không có nguồn.
4. Không được gọi tài liệu là reviewed hoặc published nếu không có bằng chứng.
5. Nếu file được sửa nhưng chưa đạt chuẩn wiki Obsidian, agent phải chuẩn hóa tối thiểu trong cùng diff, trừ khi việc đó làm tăng phạm vi quá lớn. Nếu không chuẩn hóa được, phải ghi rõ trong final report.

2. Rule cho cả file mới và file sửa

Thêm rule “Normalize Touched Markdown Files”:

Bất kỳ file .md nào được agent tạo mới hoặc chỉnh sửa đều phải được kiểm tra các tiêu chí sau:
1. Có YAML frontmatter hợp lệ nếu file là tài liệu trong docs hoặc methodology.
2. Có đúng một H1.
3. Có Purpose hoặc Summary.
4. Có Context nếu tài liệu thuộc methodology, architecture, requirement, QA, runbook hoặc design.
5. Có Related hoặc See also.
6. Có source_of_truth hoặc References nếu nội dung dựa trên tài liệu khác.
7. Có status.
8. Có internal links tới artifact liên quan.
9. Không phá code fence.
10. Không phá bảng Markdown.
11. Không phá link hiện có.
12. Không để tài liệu trở thành orphan document.

Nếu file .md là README đơn giản, changelog, license hoặc third party generated document, agent không bắt buộc phải ép toàn bộ schema, nhưng vẫn phải không làm hỏng cấu trúc Markdown.
