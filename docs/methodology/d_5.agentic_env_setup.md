# AGENTIC ENV SETUP RUNBOOK

## 1. Purpose

Thiết lập môi trường phát triển để coding agent như Codex, Claude Code, Antigravity và các agent tương tự làm việc hiệu quả, an toàn, ít tốn context, và dễ lặp lại trên máy khác.

Trong [knowledge_base.md](./knowledge_base.md), tài liệu này thuộc trụ cột "Grounding, Retrieval & Traceability với `AGENTS.md`, manifest, `kb.ps1`, `rg`" và là runbook chính cho phần dev environment.

## 2. Scope

Áp dụng cho:

1. Windows
2. Linux
3. Repo Python
4. Repo JavaScript hoặc TypeScript
5. Repo Java
6. Repo có GitHub workflow

## 3. Target toolset

- [ ]  git
- [ ]  rg
- [ ]  fd
- [ ]  bat
- [ ]  jq
- [ ]  yq
- [ ]  gh
- [ ]  uv
- [ ]  node
- [ ]  pwsh trên Windows
- [ ]  bash hoặc zsh trên Linux

## 4. Windows setup runbook

### 4.1 Prerequisites

- [ ]  Mở PowerShell hoặc Windows Terminal với quyền phù hợp
- [ ]  Có `winget` hoạt động
- [ ]  Nếu máy thuộc mạng công ty thì xác nhận `winget search` dùng được

### 4.2 Install commands

Chạy từng lệnh sau:

```powershell
winget install --id Git.Git -e
winget install --id Microsoft.PowerShell -e
winget install --id jqlang.jq -e

Nếu `rg` chưa có:

```powershell

```

Nếu `node` chưa có, tìm package phù hợp rồi cài:

```powershell
winget search node
```

### 4.4 PATH troubleshooting

Nếu tool đã cài nhưng terminal chưa nhận:

```powershell
where.exe git
where.exe pwsh
where.exe jq
where.exe yq
where.exe fd
where.exe bat
where.exe gh
where.exe uv
where.exe rg
where.exe node
```

Nếu `uv` chưa vào PATH, chạy:

```powershell
uv tool update-shell

### uv on Windows

Cài `uv`:

```powershell
winget install --id astral-sh.uv -e
```

Kiểm tra:

```powershell
winget list --id astral-sh.uv
where.exe uv
uv --version
```

Nếu `uv` chưa có trên PATH, thử thêm tạm thời:

```powershell
$env:Path += ";$env:USERPROFILE\.local\bin"
uv --version
```

Nếu chạy được, thêm vĩnh viễn vào User PATH:

```powershell
$userBin = "$env:USERPROFILE\.local\bin"
[Environment]::SetEnvironmentVariable(
  "Path",
  [Environment]::GetEnvironmentVariable("Path", "User") + ";$userBin",
  "User"
)
```

Đóng terminal và mở lại, rồi kiểm tra:

```powershell
uv --version
```

Nếu vẫn không được, cài lại bằng installer chính thức:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```
```

Sau đó mở terminal mới.

## 5. Linux setup runbook

### 5.1 Ubuntu hoặc Debian quick path

### Base tools

```bash
sudo apt update
sudo apt install -y git ripgrep fd-find bat jq nodejs npm curl
```

### Alias normalization

Trên Debian hoặc Ubuntu, `fd` có thể là `fdfind`, và `bat` có thể là `batcat`.

Nếu dùng bash:

```bash
echo 'alias fd=fdfind' >> ~/.bashrc
echo 'alias bat=batcat' >> ~/.bashrc
source ~/.bashrc
```

Nếu dùng zsh:

```bash
echo 'alias fd=fdfind' >> ~/.zshrc
echo 'alias bat=batcat' >> ~/.zshrc
source ~/.zshrc
```

### uv official installer

```bash
curl -LsSf <https://astral.sh/uv/install.sh> | sh
```

Nếu `uv` chưa vào PATH:

```bash
uv tool update-shell
```

Mở shell mới sau khi chạy.

### yq quick install

```bash
sudo apt install -y yq
```

### GitHub tooling quick install

```bash
sudo apt install -y gh
```

### 5.2 RHEL, Rocky, AlmaLinux quick path

```bash
sudo dnf install -y git ripgrep fd-find bat jq nodejs npm curl
curl -LsSf <https://astral.sh/uv/install.sh> | sh
```

`yq` và `gh` cài qua repo chuẩn của distro hoặc nguồn chuẩn nội bộ của tổ chức.

### 5.3 Arch quick path

```bash
sudo pacman -Syu --noconfirm
sudo pacman -S --noconfirm git ripgrep fd bat jq yq github tooling nodejs npm
curl -LsSf <https://astral.sh/uv/install.sh> | sh
```

## 6. Repo onboarding runbook

### 6.1 Clone or open repo

Nếu dùng GitHub tooling:

```bash
gh repo clone <owner>/<repo>
cd <repo>
```

Hoặc:

```bash

cd <repo>
```

### 6.2 Git sanity checks

```bash
git status
git branch
git remote -v
```

Pass criteria:

- [ ]  `git status` chạy bình thường
- [ ]  repo mở được
- [ ]  remote đúng

### 6.3 Search and discovery smoke test

```bash
fd AGENTS.md
fd README
fd package.json
fd pyproject.toml
fd pom.xml
rg "test|build|lint|main\\\\(" .
```

Pass criteria:

- [ ]  tìm được file chính
- [ ]  tìm được symbol hoặc text chính
- [ ]  không cần duyệt cây thư mục dài

### 6.4 Read smoke test

```bash
bat AGENTS.md
```

Nếu file không tồn tại:

```bash
bat README.md
```

Pass criteria:

- [ ]  đọc file được
- [ ]  không lỗi encoding
- [ ]  agent có thể dùng file này làm entry point

### 6.5 Structured data smoke test

Nếu có `package.json`:

```bash
cat package.json | jq .scripts
```

Nếu có workflow YAML:

```bash
yq . .github/workflows/*.yml
```

Pass criteria:

- [ ]  parse được JSON
- [ ]  parse được YAML
- [ ]  không cần đọc raw text dài để hiểu config

## 7. Runtime specific runbook

### 7.1 Python

### Discovery

```bash
fd pyproject.toml
fd uv.lock
fd requirements.txt
rg "pytest|ruff|mypy" .
```

### Setup

Nếu repo dùng `uv`:

```bash
uv sync
```

### Validate

```bash
uv run pytest -q
```

Nếu cần scope nhỏ:

```bash
uv run pytest -q path/to/test_file.py
```

Pass criteria:

- [ ]  dependency sync được
- [ ]  test nhỏ chạy được
- [ ]  agent có thể validate theo scope hẹp trước

### 7.2 Node.js hoặc TypeScript

### Discovery

```bash

```

### Install

Nếu repo dùng npm:

```bash
npm install
```

Nếu repo dùng package manager khác thì chạy đúng tool của repo.

### Validate

```bash
npm test
```

Nếu có target nhỏ hơn:

```bash
npm test -- <target>
```

Pass criteria:

- [ ]  dependency install được
- [ ]  script đọc được
- [ ]  có thể validate theo scope nhỏ trước

### 7.3 Java

- [ ]  test module hoặc test scope nhỏ chạy được

## 8. [AGENTS.md](http://agents.md/) runbook

### 8.1 Required content

`AGENTS.md` nên có các phần sau:

- [ ]  Tool priority
- [ ]  Search policy
- [ ]  Read policy
- [ ]  Diff policy
- [ ]  Edit policy
- [ ]  Validation policy
- [ ]  Structured data policy
- [ ]  Cross platform notes
- [ ]  Response style after tool usage

### 8.2 Operational rules that must exist

- [ ]  Search before read
- [ ]  Read narrow before read wide
- [ ]  Diff before reread
- [ ]  Small test before full test
- [ ]  Use `rg` before opening many files
- [ ]  Use `fd` before directory crawling
- [ ]  Use `jq` for JSON
- [ ]  Use `yq` for YAML
- [ ]  Use `gh` for GitHub tasks
- [ ]  Use `uv` for Python repos
- [ ]  Keep terminal output short in conversation
- [ ]  Prefer minimal edits

### 8.3 Quick validation

```bash
bat AGENTS.md
rg "rg|fd|jq|yq|gh|uv|git diff|search before read" AGENTS.md
```

Pass criteria:

- [ ]  `AGENTS.md` tồn tại
- [ ]  có chỉ dẫn rõ về tool usage
- [ ]  không quá mơ hồ
- [ ]  không yêu cầu tool không tồn tại

## 12. Troubleshooting runbook

### 12.1 Tool installed but not found

Windows:

```powershell
where.exe <tool>
```

Linux:

```bash
which <tool>
```

Fix:

- [ ]  mở terminal mới
- [ ]  kiểm tra PATH
- [ ]  chạy lại shell init
- [ ]  xác nhận binary đúng

### 12.2 GitHub tooling not authenticated

```bash
gh auth login
gh auth status
gh auth setup-git
```

### 12.3 Agent reads too many files

Fix:

- [ ]  cập nhật `AGENTS.md`
- [ ]  ép `rg` trước `bat`
- [ ]  ép `fd` trước duyệt cây thư mục
- [ ]  ép diff trước reread
- [ ]  ép summary ngắn sau command group

### 12.4 Agent dumps long logs

Fix:

- [ ]  chỉ báo exit status
- [ ]  chỉ báo summary pass hoặc fail
- [ ]  chỉ báo lỗi đầu tiên quan trọng
- [ ]  chỉ báo next action
- [ ]  không paste raw output dài nếu không được yêu cầu
